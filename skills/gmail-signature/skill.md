## OAuth Scope
Requires `gmail.settings.basic` — separate from the eMMA bidder's `gmail.compose` scope.
Running the script opens a browser for one-time approval; no token is saved (re-auth each run).

## How to Run
```
python C:\Users\Pinko\claudesandbox\set_gmail_signature.py
```
Browser opens → approve Gmail Settings access → done. Check Gmail → Settings → General → Signature.

## How to Update the Signature
1. Edit `SIGNATURE_HTML` in `set_gmail_signature.py`
2. Run the script
3. Re-auth if prompted

## Key Dependencies
Already installed via eMMA bidder env:
- `google-auth-oauthlib`
- `google-api-python-client`

## Common Issues
- **Token scope error:** The existing emma-bidder token.json only has `gmail.compose`. The script re-auths fresh for `gmail.settings.basic` — this is expected.
- **Logo not showing:** Check that `proles-logo-white.svg` is deployed on prolesconsulting.com. Run `railway up` from the proles-consulting directory.
- **QR code not loading:** The QR is generated by `api.qrserver.com` on-the-fly. No action needed unless that service is down.
